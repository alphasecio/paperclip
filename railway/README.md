# Paperclip on Railway

Deploy [Paperclip](https://github.com/paperclipai/paperclip) — the open-source AI agent orchestration platform — on [Railway](https://railway.com/?referralCode=alphasec) with a single click. Supports both embedded PostgreSQL (single service) and external PostgreSQL (two services).

See [this](https://alphasec.io/stop-hiring-start-orchestrating-running-an-ai-agent-company-with-paperclip-on-railway) companion blog post for a step-by-step walkthrough.

## Prerequisites

- A [Railway](https://railway.com/?referralCode=alphasec) account
- An API key from at least one LLM provider (Anthropic, OpenAI, or Gemini)

## Configuration

Set the following environment variables in the Railway dashboard after deploying:

| Variable | Description |
|---|---|
| `BETTER_AUTH_SECRET` | Auto-generated by Railway on deploy |
| `PAPERCLIP_AGENT_JWT_SECRET` | Auto-generated by Railway on deploy |
| `PAPERCLIP_PUBLIC_URL` | Auto-set from Railway public domain |
| `PAPERCLIP_ALLOWED_HOSTNAMES` | Auto-set from Railway public domain |
| `DATABASE_URL` | Optional — leave unset to use embedded PostgreSQL, or wire to a Railway Postgres addon |
| `PAPERCLIP_AUTH_DISABLE_SIGN_UP` | Set to `true` after creating your admin account |
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude agents |
| `OPENAI_API_KEY` | OpenAI API key for GPT agents |
| `GEMINI_API_KEY` | Gemini API key for Gemini agents via opencode |

## Deployment modes

**Embedded PostgreSQL (single service)** — leave `DATABASE_URL` unset. Paperclip runs with its built-in database. Lower cost, suitable for evaluation and small teams.

**External PostgreSQL (two services)** — add a Railway Postgres addon to your project and set `DATABASE_URL` to `${{Postgres.DATABASE_URL}}`. More robust, recommended for production use.

## Post-Deploy

1. Use the bootstrap CEO invite URL printed in the service logs to create your admin account:
   - Go to your active deployment → **View logs** → search for `invite` in **Deploy logs**
3. Create the admin user, and set up the Company with initial agent and onboarding task
4. For single-user mode, set `PAPERCLIP_AUTH_DISABLE_SIGN_UP=true` in service Variables and redeploy

## Agents

Paperclip's built-in adapters work out of the box with environment variables:

| Provider | Adapter | Required Variable |
|---|---|---|
| Anthropic | `claude_local` | `ANTHROPIC_API_KEY` |
| OpenAI | `opencode_local` | `OPENAI_API_KEY` |
| Gemini | `opencode_local` | `GEMINI_API_KEY` |

For Gemini, select `opencode_local` as the adapter in Paperclip's agent settings and choose your Gemini model (e.g. `gemini-2.5-flash`). The `gemini_local` adapter requires interactive OAuth and will not work in a Railway container.
